Python Data Processing & CI/CD Project

Project Goal Overview

This project addresses a common scenario in data processing and continuous integration. The primary objectives are to:

  • • Fix a non-trivial error in the execute.py script.
  • • Ensure the script runs on Python 3.11+ with Pandas 2.3.
  • • Convert data.xlsx to data.csv and integrate this into the project.
  • • Implement a robust GitHub Actions push workflow (.github/workflows/ci.yml) that includes linting (Ruff), script execution, and publishing the output via GitHub Pages.

Solution Highlights

1. execute.py: The Fix

The execute.py script was updated to ensure robust data handling, specifically addressing a potential type conversion error. It now explicitly converts the 'Value' column to a numeric type, coercing errors to NaN, and then dropping rows with invalid values before performing its analysis.
